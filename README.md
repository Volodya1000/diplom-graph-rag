
# 🧠 Дипломный проект Ontology Graph RAG

Система анализа документов на базе графа знаний (Neo4j) и локальных LLM (Ollama).
Проект позволяет извлекать сущности и связи из PDF-документов, строить граф знаний, находить сообщества и отвечать на вопросы с цитированием источников. Потому что просто хранить PDF в папке людям уже скучно.

<a id="установка-и-настройка"></a>
## 🛠 Установка и настройка

Проект использует менеджер пакетов [uv](https://github.com/astral-sh/uv).

### 1. Установка зависимостей

Для создания окружения и установки всех зависимостей:

```bash

```

<a id="инициализация-и-онтология"></a>
## Инициализация и онтология

**1. Инициализация базового T-Box:**

```bash
uv run main.py seed-tbox
```
<details>
<summary><b>Показать дополнительные параметры (вывод классов, принудительное обновление)</b></summary>

Инициализация T-Box с выводом классов и отношений:
```bash
uv run main.py seed-tbox --show
```

Принудительное обновление T-Box:
```bash
uv run main.py seed-tbox --force
```

</details>

**2. Импорт онтологии из TTL-файла:**

```bash
uv run main.py import-ontology alpha_ontology.ttl
```

**3. Экспорт онтологии:**

```bash
uv run main.py export-ontology -o data/ontology/my_ontology.ttl
```
<details>
<summary><b>Показать дополнительные параметры (перезапись файла)</b></summary>

Экспорт с принудительной перезаписью файла:
```bash
uv run main.py export-ontology -o ontology.ttl --force
```

</details>

<a id="работа-с-документами"></a>
## Работа с документами

**4. Индексация документа:**

```bash
uv run main.py ingest устав.pdf
```

**5. Информация о документе (чанки, сущности, триплеты):**

```bash
uv run main.py doc-info устав.pdf
```

<a id="построение-графа"></a>
## Построение графа

**6. Сборка сообществ графа:**

```bash
uv run main.py build-communities
```
<details>
<summary><b>Показать дополнительные параметры (выбор алгоритма, минимальный размер)</b></summary>

Сборка сообществ с алгоритмом Leiden:
```bash
uv run main.py build-communities --algo leiden
```

Сборка сообществ с минимальным размером (например, от 3 узлов):
```bash
uv run main.py build-communities --algo leiden --min-size 3
```

</details>

<a id="работа-с-системой"></a>
## Работа с системой

**7. Задать вопрос:**

```bash
uv run main.py ask "Ваш вопрос?"
```
<details>
<summary><b>Показать дополнительные параметры (режимы поиска, лимит источников)</b></summary>

**Режимы поиска:**
Доступные режимы: `local`, `global`, `local_ppr`, `hybrid`.

Пример (гибридный поиск):
```bash
uv run main.py ask "Ваш вопрос?" --mode hybrid
```

**Количество источников:**
Ограничение количества используемых документов (top-k):
```bash
uv run main.py ask "Ваш вопрос?" --top-k 15
```

</details>

<a id="очистка-базы"></a>
## Очистка базы

**8. Полная очистка Neo4j:**

```bash
uv run main.py drop-db
```
Удаляет документы, сущности, отношения, communities и T-Box.
Команда для тех редких моментов, когда «давай просто всё удалим и начнём заново» действительно кажется хорошей идеей.

<a id="запуск-rest-api"></a>
## 🌐 Запуск REST API

Для запуска API-сервера в режиме разработки с доступностью извне:

```bash
uv run uvicorn src.presentation.api.main:app --reload --host 0.0.0.0 --port 8000
```

* **Swagger UI**:[http://localhost:8000/docs](http://localhost:8000/docs)
* **Хост**: `0.0.0.0` позволяет принимать запросы не только с `localhost`, но и из локальной сети или Docker-контейнеров.

<a id="краткая-структура-проекта"></a>
## 📁 Краткая структура проекта
* `src/domain/` — бизнес-логика и модели.
* `src/application/` — юз-кейсы и пайплайны.
* `src/infrastructure/` — реализация (LLM, Embeddings, Docling, Neo4j репозитории).
* `src/presentation/` — CLI (typer) и API (fastapi).
* `tests/` — unit и integration тесты.

---
**Примечание:** Перед запуском `ingest` или `ask`, убедитесь, что ваш `.env` файл содержит корректные параметры подключения к Neo4j, а Ollama запущена и доступна.
